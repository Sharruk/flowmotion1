Experiment 6: Demonstrate continuous integration and development with GitLab
Scenario:
You are required to design, configure, and analyze a CI/CD pipeline using GitLab to
automate build, testing, and deployment processes, and evaluate its effectiveness
based on observed performance metrics.
Marks Distribution:
• Pipeline Setup & Evaluation – 40
• Demo / Defense – 40
• Documentation – 20
Task:
Pipeline Setup and Configuration (K3 | CO4 & CO5 | PI:1.3.1, 2.1.3, 3.4.1, 3.4.3)
Students should:
• Apply CI/CD concepts to configure a GitLab CI pipeline using gitlab-ci.yml
• Implement automated stages such as build, test, and deploy
• Integrate basic testing or deployment scripts
• Execute the pipeline and verify successful stage execution
• Document pipeline configuration with screenshots and logs
Pipeline Analysis and Evaluation (K4 | CO4 & CO5 | PI: 2.2.3, 2.2.4, 5.1.2 & 3.2.1)
Students should:
• Analyze pipeline execution results such as build success, test pass/fail, and
deployment status
• Examine pipeline efficiency using execution time, failure points, and logs
• Analyze reliability and consistency across multiple pipelines runs
• Identify limitations or bottlenecks in the configured pipeline
Improvement Recommendations (K4 | CO5 | PI: 5.1.2)
Students should:
• Propose improvements such as parallel jobs, caching, or additional test stages
• Justify recommendations based on observed pipeline behavior
• Discuss how the proposed changes improve efficiency or reliability
Demo and Defense (K3 & K4 | CO4 & CO5)
Students should:
• Demonstrate the working CI/CD pipeline
• Explain design decisions, stage configuration, and scripts used
• Defend pipeline choices based on performance observations and analysis
